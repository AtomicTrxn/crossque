Accessibility

Color Blind Mode — Option D

Option D replaces the green/red verification pair with a blue/orange pair that survives deuteranopia and protanopia, and adds ✓/✗ symbols as a second independent signal covering achromatopsia. Only two letter-color tokens change; all position backgrounds, the word-completion background, and the revealed background are unchanged.

Implementation rule

The CB letter-color override applies to all verified cells — including cells inside completed words. The word-completion background (#C8E6C9) is unchanged; it simulates to neutral gray under deuteranopia and remains distinct from all position backgrounds. No token changes are needed for word-completion beyond the existing CB letter override.

Five-layer semantic model

Every visual layer carries exactly one semantic concern. Layers are fully independent — a cell's background and its letter color are set by different rules and never conflict.

Background priority stack

When a cell qualifies for multiple position states, the highest-priority background wins. Letter color and symbol are always determined by verification state regardless of which background wins.

Word-level completion

✓ No extra token changes needed

Word-completed cells keep the green background (#C8E6C9) in CB mode. Under deuteranopia this simulates to neutral gray (~#CBCBCB) — distinguishable from all position backgrounds. The CB letter override makes the letter blue + ✓, identical to any other verified-correct cell. The background difference is what distinguishes "this word is complete" from "this cell is individually checked."

Puzzle-level completion (full-screen green overlay) is a terminal state and requires no CB override — it is never concurrent with active solving UI.

Symbol specification

  • Correct symbol: ✓ (U+2713), 8–9px, top-right of cell, color matches gridCorrectLetter CB value
  • Incorrect symbol: ✗ (U+2717), 8–9px, top-right of cell, color matches gridIncorrectLetter CB value
  • Position: top: 2px, right: 3px — mirrors the clue number at top: 2px, left: 3px
  • Applies to: all verified cells in any position state (empty, active, completed, focused)
  • Does not apply to: revealed cells — provenance is not a verification state
  • Dynamic color: declare CB overrides as fixed resource values to prevent Android system dynamic color from remapping them
